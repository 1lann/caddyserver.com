Write Anytime. Preview Anywhere.

Caddy is a web server that makes it easy for writers to work on their articles offline. Whether it's HTML, Markdown, or even a WordPress blog, Caddy can serve it up to your browser. Caddy can be used both on your own computer and your public web host.

Complements Any Workflow

When you sit down to write, start up Caddy and leave it running for as long as you want to be able to preview your work in your web browser. It doesn't matter whether you're writing in WordPress, Drupal, or Markdown files—Caddy supports it.

Caddy can also be deployed to a public web host to serve your site to the world. Caddy works the same on your server as it does on your computer so the transition is painless.

How It Works

Place a Caddyfile with your site. Run caddy on the command line, and leave it running while you work on your site. While Caddy is running, you can open your site in a web browser, allowing you to see it just as if it were published on a public web host for everyone to see.

Running on your public server is the same: upload your site along with a Caddyfile, run caddy, and Caddy will do what it does best. You don't have to restart Caddy when you make changes to your site's files.

If you want to use your own domain name on your computer (like "my.blog"), all you need to do is add a line to your hosts file.

Sample Caddyfiles

my.blog:8080 fastcgi / 127.0.0.1:9000 php
  • Site's URL is my.blog:8080 ("my.blog" was added to hosts file)
  • Since the blog is powered by WordPress, proxy all requests (under the path /) to FastCGI with preset configurations for serving PHP sites

http://example-site.com markdown /articles { css /articles.css js /interactions.js }
  • Site's domain is "example-site.com" - this Caddyfile is serving a production website
  • Markdown files in the /articles folder are rendered as HTML with custom CSS and JS files
