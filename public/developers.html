More dev, less ops

Caddy is a capable alternative to other web servers and has only a fraction of the complexity.

Why Caddy?

  • Easier than nginx
  • Faster than httpd
  • No dependencies
  • Portable, pre-compiled binary
  • HTTP/2
  • The Caddyfile
  • Extensible middleware layer
  • Templates
  • Easy to deploy
  • Multi-core
  • ... really, just try it.

Virtual Hosts

Caddy has full virtual host support, meaning it can serve multiple sites on the same address. It even supports Server Name Indication to serve multiple HTTPS sites on the same port.

Extra Goodies

Execute commands at startup and shutdown; gzip; custom headers; custom templates for directory browsing and Markdown files; rewrites and redirects; proxy; basic auth; error pages, and more.

A Joy to Configure

A major motivation for Caddy was the difficulty of configuring other web servers. Stop banging on conf files with a hammer and get on some fresh turf with Caddy.

Use Caddy in your programs

Caddy as a Service

Caddy can be used in your own Go programs simply by importing the caddy package. This is ideal for programs with Web endpoints that need a high degree of configurability. It's easy to set up and use - just a few lines of code.

Extend Caddy

Anyone can write middleware for Caddy (or adapt existing middleware) to add functionality and integrate with 3rd-party services. Such extensions are available as plugins from the Download page already!

Usage Example

You can start, restart, and stop Caddy very easily: 

import "github.com/mholt/caddy/caddy"

caddyfile := caddy.CaddyfileInput{ /* ... */ }

// error checking omitted for brevity
err := caddy.Start(caddyfile)
err = caddy.Restart(nil)
err = caddy.Stop()

Compatible with your workflow

Caddy is so simple to use, it fits snugly into your existing workflow.

Take Caddy Anywhere

Caddy is cross-platform and has no dependencies. It's a single binary file that's easy to deploy: just copy it.

Portable Configurations

The Caddyfile, which configures Caddy, is completely separate from Caddy itself. This makes it easy to adapt to any workflow.
