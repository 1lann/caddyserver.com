cms
Addon

Addon
This directive is a Caddy extension. Questions should be directed to its maintainer. github.com/hacdias/caddy-cms

cms allows you to deploy your static website and enjoy of a CMS-like admin interface where you can edit content, schedule posts, upload files, etc. The admin interface will be available at /admin/ on your website.

By default, it uses Hugo - which is bundled with the plugin so you don't have to install it separately - but you can use any other static website generator.

Requirements: if you wan't to use Hugo (which is the default and the recommended), you only need to have an already created functional website for Hugo. If you want to use an external static website generator, is must be in your PATH.

Syntax

cms { styles file args args... command name content folder # default is content }
  • file is the relative path to public folder of the admin UI styles. These styles won't replace the defaults, they will be added.
  • args are the command line arguments you usually set when using the static website generator.
  • command is the command name of the static website generator you want to use. If you don't set this directive, Hugo will be used.
  • content is the folder where the main content of the website is placed. It affects the 'Content' link on navigation bar. By default it is 'content', but if you are using a static website generator which is not Hugo, you will probably need to change this variable.

All of the directives above are optional.

To schedule a post, its date field must be in '2006-01-02 15:04:05-07:00' format.

hugo plugin should be used with root, basicauth and errors middleware to have the best experience. See the examples to know more.

Examples

A simple configuration to use with Hugo static website generator:

localhost:8080 # website URL and port root public # the folder where Hugo generates the website basicauth /admin user pass # protect the admin area using HTTP basic auth errors { 404 404.html # errors page (relative to root folder) } cms

A functional Jekyll configuration:

localhost:8080 root public basicauth /admin user pass errors { 404 404.html } cms { command jekyll content _posts }
