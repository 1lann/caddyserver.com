hugo
Addon

Addon
This directive is a Caddy extension. Questions should be directed to its maintainer. github.com/hacdias/caddy-hugo

hugo allows you to deploy your static Hugo website and enjoy of a CMS-like admin interface where you can edit content, schedule posts, upload files, etc.

This plugin will regenerate your website everytime you publish some content page or when you schedule a post.

The admin interface will be available at '/admin/' on your website.

Requirements: the only requirement you need is to have a functional Hugo website already created. You don't need to have Hugo installed on your computer because it is bundled with this plugin.

Syntax

hugo { styles file flags flags... }
  • file is the relative path to public folder of the admin UI styles. These styles won't replace the defaults, they will be added.
  • flags are the Hugo flags (those which can be set in the command line) and they must follow one of these syntaxes: -f=value or --flag=value

To schedule a post, its "date" field must be in "2006-01-02 15:04:05-07:00" format.

hugo plugin should be used with root, basicauth and errors middleware to have the best experience. See the examples to know more.

Examples

A simple Hugo website:

localhost:8080 # website URL and port root public # the folder where Hugo generates the website basicauth /admin user pass # protect the admin area using HTTP basic auth errors { 404 404.html # errors page (relative to root folder) } hugo

A fully functional website with all of the functions:

localhost:8080 # website URL and port root public basicauth /admin user pass errors { 404 404.html } hugo { styles public/css/admin.css # additional admin styles flags --theme="mytheme" # hugo flags }
