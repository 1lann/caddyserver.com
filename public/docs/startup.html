startup

startup executes a command when the server begins. This is useful for preparing to serve a site by running a script or starting a background process like php-fpm. (Also see shutdown.)

Each command that is executed at startup is blocking, unless you suffix the command with &, which will cause the command to be run in the background. The output and error of the command go to stdout and stderr, respectively. There is no stdin.

Syntax

startup command
  • command is the command to execute

The command may be passed arguments/flags, but remember that if there are any spaces, the command and its arguments must enclosed with quotes. See the example.

Examples

Start php-fpm before the server starts listening:

startup "/etc/init.d/php-fpm start"
