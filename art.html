When you load your site with a "real" domain name in your browser, you can do more than if it was loaded as a file.

Runs the same everywhere

Caddy runs the same on your computer as it does on your public web server. Configuration and setup never get in your way.

So—wait, what is it exactly?

Caddy is a web server. Every website you go to comes from a web server somewhere on the Internet. But what if you could load websites that you're designing on your own computer without making it public?

Caddy lets you do just that. Loading a page by going to File → Open has some limitations, but using Caddy to serve your site with a domain name sets your site free.

Sample Caddyfiles

localhost ext .html errors { 404 notfound.html 503 serviceunavail.html }
  • Site's URL is the default "localhost"
  • Clean URLs by trying .html files
  • Allows you to design and test error pages for Not Found and Service Unavailable errors
http://julia.design root /Users/julia/Sites/julia-design markdown { css /resources/css/articles.css js /resources/js/interactions.js }
  • Site's domain is "julia.design" - works by adding "127.0.0.1 julia.design" to hosts file
  • The site's files are located in Julia's /julia-design folder
  • Markdown files are rendered as HTML with Julia's CSS and JS files
